# ⚠️ Frontend Deployment Notice

**ВАЖНО:** Фронтенд больше не деплоится отдельно!

Фронтенд теперь является частью монолитного приложения и деплоится вместе с бэкендом.

## Архитектура

- **Backend** (`backend/Dockerfile`) - монолитное приложение, которое:
  - Собирает фронтенд во время Docker build
  - Раздает статические файлы фронтенда через Express
  - Обрабатывает API запросы

- **Bot** (`bot/Dockerfile`) - отдельный сервис для Telegram бота

## Railway Configuration

В Railway должны быть настроены только **2 сервиса**:

1. **Backend** - использует `backend/Dockerfile` (монолит)
2. **Bot** - использует `bot/Dockerfile`

**НЕ создавайте отдельный сервис для Frontend!**

Если у вас есть отдельный Frontend сервис в Railway:
1. Удалите его из проекта Railway
2. Убедитесь, что только Backend и Bot сервисы активны

## Проверка

После деплоя Backend должен:
- Отвечать на `/api/*` запросы (API)
- Отдавать фронтенд на всех остальных путях (SPA routing)
